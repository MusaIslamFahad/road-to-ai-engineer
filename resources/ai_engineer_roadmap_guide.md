# AI Engineer (Generalist) — Career Roadmap Guide

This guide provides a complete overview of the **AI Engineer (Generalist)** career path: what the role demands, how to prioritize the 23 modules in this repo, which projects to build, and what interviewers look for.

---

## Role Overview

**AI Engineer (Generalist)** is the broadest technical AI role. Unlike specialists, a Generalist AI Engineer can:
- Design and train models across multiple domains (vision, language, audio, graphs)
- Build and deploy production AI systems end-to-end
- Rapidly prototype in any AI domain and hand off to specialists when needed
- Bridge research and engineering — translate papers into products

**Typical responsibilities**:
- Build ML/DL models for varied business problems
- Integrate LLMs and GenAI features into products
- Design data pipelines, experiment frameworks, and monitoring
- Deploy models to cloud with proper MLOps
- Mentor junior team members across domains

**Salary range (2025)**: $140,000–$350,000+ in the US depending on company/level/location

## Skills Checklist

### Machine Learning
- [ ] Supervised: regression, classification (logistic, trees, SVM, KNN, Naive Bayes)
- [ ] Unsupervised: K-Means, DBSCAN, hierarchical, PCA, t-SNE, UMAP, anomaly detection
- [ ] Ensemble: Random Forest, XGBoost, LightGBM, CatBoost, stacking
- [ ] Feature engineering and sklearn Pipelines
- [ ] Cross-validation, hyperparameter tuning (Optuna), model evaluation

### Deep Learning
- [ ] Neural networks from scratch (NumPy)
- [ ] PyTorch: custom training loops, DataLoader, autograd
- [ ] Optimizers, schedulers, regularization, batch norm
- [ ] TensorFlow/Keras functional API

### Computer Vision
- [ ] CNN architectures: ResNet, EfficientNet, ConvNeXt
- [ ] Object detection: YOLO v8, Faster R-CNN; mAP evaluation
- [ ] Segmentation: U-Net, Mask R-CNN
- [ ] GANs, VAEs, Stable Diffusion
- [ ] Vision Transformers (ViT)

### NLP & LLMs
- [ ] Transformer architecture (self-attention, multi-head, positional encoding)
- [ ] BERT fine-tuning for classification/NER/QA
- [ ] Hugging Face Trainer API
- [ ] LoRA/QLoRA fine-tuning
- [ ] RAG: chunking, embedding, retrieval, re-ranking, evaluation (RAGAS)

### Generative AI
- [ ] Prompt engineering patterns
- [ ] LangChain / LCEL; memory; tools
- [ ] AI agents: ReAct, tool use, planning
- [ ] Multi-agent: CrewAI, AutoGen, LangGraph
- [ ] MCP and A2A protocols
- [ ] LLM observability: LangSmith, Langfuse

### Production
- [ ] FastAPI: async routes, validation, streaming, auth
- [ ] Docker: Dockerfile, docker-compose, multi-stage builds
- [ ] AWS: SageMaker, ECS, ECR, Lambda
- [ ] MLflow: tracking, model registry
- [ ] DVC: data versioning
- [ ] GitHub Actions: CI/CD for ML
- [ ] Model monitoring: drift detection, alerting

### Advanced Domains
- [ ] RL: Q-Learning, DQN, PPO, multi-agent (one working agent minimum)
- [ ] GNNs: GCN or GAT on a real graph dataset
- [ ] Audio: Whisper transcription, audio classification, TTS basics
- [ ] Time series: ARIMA, LSTM forecaster, walk-forward validation

## Interview Preparation

### Common Question Types

#### ML Fundamentals
- Explain the bias-variance tradeoff with an example
- When would you use L1 vs. L2 regularization?
- How does XGBoost differ from Random Forest?
- What is data leakage and how do you prevent it?
- Explain cross-validation and when to use each type

#### Deep Learning
- Explain backpropagation — step by step through one layer
- Why do we need batch normalization?
- Compare Adam vs. SGD with momentum — when use each?
- What are vanishing gradients and how do you fix them?

#### Computer Vision
- Explain how a convolutional layer works
- What is transfer learning and when do you use it?
- How does YOLO detect multiple objects in one pass?
- What is the difference between semantic and instance segmentation?

#### NLP & LLMs
- Explain self-attention from scratch
- What is the difference between BERT and GPT architectures?
- How does RAG work? What are its failure modes?
- When would you fine-tune vs. use RAG vs. prompt engineer?

#### Production
- How do you monitor a model in production?
- What is A/B testing and when do you use it vs. multi-armed bandits?
- Explain CI/CD for ML — what does your GitHub Actions pipeline look like?
- How would you design a recommendation system for 1M users?

### System Design Questions (AI Focus)
- Design a real-time fraud detection system
- Design a document Q&A system (RAG architecture)
- Design a ML platform for 100 data scientists
- Design an image moderation system for 10M images/day
- Design a recommendation engine for an e-commerce platform

## Building Your Portfolio

A strong AI Engineer portfolio demonstrates **breadth** across domains and **depth** in at least 2–3 areas.

### Recommended Portfolio Structure

1. **Tabular ML project** (beginner–intermediate): clean dataset, proper CV, feature engineering, deployed API
2. **Computer Vision project** (advanced): custom object detection or image generation
3. **NLP/LLM project** (advanced): fine-tuned transformer OR production RAG system
4. **End-to-End pipeline** (advanced): full MLOps with Docker, CI/CD, monitoring
5. **Capstone** (portfolio-grade): choose one from [18-projects-advanced/capstones/](../18-projects-advanced/)

### What to Include in Each Project
- Clear README with problem statement, approach, and results
- Clean, well-commented code
- Model evaluation with proper metrics
- At minimum a Streamlit demo or deployed API endpoint
- Recorded demo video (optional but impressive)

### Where to Host
- **GitHub**: all code with good READMEs
- **Hugging Face Spaces**: interactive Gradio/Streamlit demos (free GPU)
- **Personal website**: project showcase with links

---

**Remember**: The AI Engineer path is a marathon. Focus on depth of understanding over speed of completion. Build real projects at every phase — portfolio > certificates.
