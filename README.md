## Why AI Engineer (Generalist)?

The **AI Engineer** is the broadest and most versatile role in the AI field. Unlike specialists who focus on a single domain, a Generalist AI Engineer has:

- **Full-spectrum ML** — Classical ML, ensemble methods, feature engineering
- **Deep Learning mastery** — Neural networks, CNNs, RNNs, Transformers
- **Computer Vision** — Object detection, segmentation, generative models
- **Natural Language Processing** — Transformers, fine-tuning, RAG
- **Generative AI & LLMs** — Prompt engineering, AI agents, multi-agent systems
- **Reinforcement Learning** — Q-Learning, DQN, policy gradients
- **Graph Neural Networks** — GCNs, GATs, graph-based reasoning
- **Audio & Speech** — ASR, TTS, audio classification
- **Production & MLOps** — Deployment, monitoring, CI/CD, cloud

**Perfect for**: Professionals who want to work across the full breadth of AI, build end-to-end AI products, and adapt to any AI problem domain.

## Phase 0: Foundation (Prerequisites)

**Goal**: Build the mathematical and programming foundation needed for all AI domains.

- **00-prerequisites**
  - **Python Programming**
    - Variables, operators, control flow, strings, lists, tuples, sets, dictionaries; `import` and modules
    - Functions and functional programming; `*args` / `**kwargs`; nested functions and scope
    - OOP: classes and objects; magic methods; inheritance, polymorphism, encapsulation, abstraction
    - Iterators and generators; lambda functions; `map`, `filter`, `reduce`
    - File handling: text and binary I/O; JSON and pickle; context managers (`with`)
    - Decorators; namespaces and LEGB scope; exception handling
    - Time Complexity & Big O notation
    - GUI Development with tkinter
  - **Essential Mathematics**
    - Linear Algebra (vectors, matrices, tensors, eigenvalues, SVD)
    - Statistics and Probability (distributions, hypothesis testing, Bayes' theorem)
    - Calculus (derivatives, gradients, chain rule, optimization)
    - Computational Math: Python code alongside theory
  - **Environment Setup**: Python, Jupyter, virtual environments, Git & GitHub
  - **Project**: Movie script generator

---

## Phase 1: Data Fundamentals

**Goal**: Master data manipulation, visualization, and the full Python data science stack.

- **01-python-for-data-science**
  - **NumPy**: `ndarray`, dtypes, indexing/slicing, broadcasting, vectorized operations, linear algebra
  - **Pandas**: Series, DataFrame, `loc`/`iloc`, `GroupBy`, merge/join/concat, MultiIndex, pivot tables
  - **Polars**: High-performance data manipulation (10–100× faster than Pandas for large datasets)
  - **Dask**: Parallel Pandas for larger-than-memory datasets
  - **Matplotlib / Seaborn**: Line, scatter, histogram, bar, pie charts; publication-quality plots
  - **Plotly & Dash**: Interactive visualizations and web applications
  - **Streamlit**: Interactive dashboards and ML applications
  - **Flask**: Web applications and REST APIs
  - **Tableau**: Professional data visualization
  - **EDA, Data Sources, Regular Expressions, Advanced Data Wrangling**
  - **ETL with AWS RDS**: Extract, Transform, Load pipelines
  - **Advanced Selenium Web Scraping**: Dynamic content scraping

---

## Phase 2: Machine Learning Basics

**Goal**: Understand core ML concepts, algorithms, and evaluation methodology.

- **02-introduction-to-ml**
  - ML types: Supervised, Unsupervised, Reinforcement Learning
  - ML workflow and best practices; batch vs. online updates; instance vs. model-based learners
  - Descriptive statistics; population vs. sample; data types in ML
  - **Project**: First end-to-end ML model

- **03-supervised-learning-regression**
  - Linear regression (simple and multiple); evaluation metrics (MSE, RMSE, MAE, R²)
  - Polynomial regression; regularization (Ridge, Lasso, ElasticNet)
  - Gradient descent: batch, stochastic, and mini-batch
  - Statistical regression analysis (statsmodels: F-statistic, p-values, confidence intervals)

- **04-supervised-learning-classification**
  - Logistic regression, K-Nearest Neighbours, Naive Bayes
  - Decision Trees, Random Forests, SVM (multiclass strategies)
  - Evaluation: Accuracy, Precision, Recall, F1, ROC-AUC
  - Bias auditing and fairness (Fairlearn)

- **05-model-evaluation-optimization**
  - Train/Validation/Test Split; Cross-Validation (K-Fold, Stratified, Time Series Split)
  - ROC and PR curves; data leakage detection and prevention
  - Hyperparameter Tuning: Grid Search, Random Search, Bayesian Optimization (Optuna)
  - Bias-Variance Tradeoff; Learning Curves; Model Calibration

---

## Phase 3: Advanced Supervised Learning

**Goal**: Explore ensemble methods, feature engineering, and advanced ML techniques.

- **06-ensemble-methods**
  - Random Forest and bagging
  - Boosting: AdaBoost, Gradient Boosting, XGBoost, LightGBM, CatBoost
  - Stacking and voting ensembles

- **07-feature-engineering**
  - Feature Selection and Transformation
  - Categorical encoding: One-Hot, Label, Target, Frequency, WOE
  - Feature Scaling and Normalization
  - Dimensionality Reduction (PCA)
  - Advanced Discretization; sklearn Pipelines and ColumnTransformer

---

## Phase 4: Unsupervised Learning

**Goal**: Make sense of unlabeled data through clustering, dimensionality reduction, and anomaly detection.

- **08-unsupervised-learning**
  - K-Means, hierarchical (Ward, complete, average), density-based (DBSCAN, HDBSCAN)
  - Clustering validation: silhouette score, elbow method
  - Dimensionality Reduction: PCA, t-SNE, UMAP, SVD
  - LDA (supervised dimensionality reduction) alongside PCA
  - Anomaly Detection: Isolation Forest, One-Class SVM, LOF
  - Association rules: Apriori, market-basket analysis

---

## Phase 5: Deep Learning Fundamentals

**Goal**: Build deep learning intuition that survives new architectures.

- **09-neural-networks-basics**
  - From linear limits to MLPs: activations, loss functions, backpropagation as chain rule
  - Optimizers: SGD family through Adam, AdaGrad, RMSprop
  - Regularization: Dropout, Batch Normalization, L1/L2
  - Weight initialization; vanishing/exploding gradients; training diagnostics

- **10-deep-learning-frameworks**
  - PyTorch tensors and autograd; custom training loops
  - Keras ergonomics; TensorFlow 2.x
  - Saving models; deployment-friendly formats (ONNX, TorchScript)
  - GPU acceleration and mixed precision training

---

## Phase 6: Specialized Deep Learning

**Goal**: Master Computer Vision and Natural Language Processing — the two core specialized AI domains.

### Computer Vision

- **11-computer-vision**
  - Convolutions, geometry (padding/stride), modern CNN families
  - Classic architectures: LeNet, AlexNet, VGG, ResNet, EfficientNet
  - Object Detection: YOLO, R-CNN, Faster R-CNN, SSD
  - Segmentation: Semantic (FCN, DeepLab), Instance (Mask R-CNN)
  - Transfer learning from strong backbones; data augmentation
  - Generative Models: GANs, VAEs, Diffusion Models, Stable Diffusion
  - Vision Transformers (ViT) and their connections to NLP

### Natural Language Processing

- **12-natural-language-processing**
  - Text preprocessing; tokenization; word embeddings (Word2Vec, GloVe, FastText)
  - RNNs → LSTMs/GRUs → bidirectional stacks
  - Self-attention mechanism; encoder-decoder architecture
  - BERT, GPT, T5, and the Transformer ecosystem
  - Hugging Face Pipelines, Datasets, and Trainer API
  - Fine-tuning pre-trained models; LoRA and parameter-efficient fine-tuning
  - RAG (Retrieval Augmented Generation) fundamentals

## Branch: Time Series Analysis

*Can be taken in parallel with or after Phase 5–6*

- **15-time-series-analysis**
  - Trend, seasonality, and stationarity; ACF/PACF plots
  - Classical models: ARIMA, SARIMA, SARIMAX, Exponential Smoothing
  - ML for time series: feature engineering, lag features, rolling statistics
  - Deep sequence models: LSTM, GRU, Temporal Convolutional Networks
  - Evaluation respecting time ordering: walk-forward validation
  - Forecasting frameworks: statsmodels, Prophet, NeuralForecast

## Phase 7: Generative AI & Modern LLM Applications

**Goal**: Work *with* foundation models — compose, deploy, and evaluate production GenAI systems.

- **25-generative-ai-llms**
  - Prompt engineering patterns: zero-shot, few-shot, chain-of-thought, ReAct
  - Embeddings and vector search; vector databases (Pinecone, ChromaDB, FAISS, Weaviate)
  - RAG pipelines: chunking, retrieval, re-ranking, generation
  - AI Agents: tool use, planning, memory, reflection
  - Multi-agent systems: CrewAI, AutoGen, LangGraph orchestration
  - LangChain and LlamaIndex frameworks
  - Model Context Protocol (MCP) and Agent-to-Agent (A2A) communication
  - LLM evaluation: RAGAS, hallucination detection, benchmarking
  - Fine-tuning LLMs: LoRA, QLoRA, RLHF basics
  - Cost optimization and LLM inference serving

## Phase 7.5: Essential Data Science Skills

**Goal**: Critical skills for real-world AI applications — best learned in parallel.

- **19-sql-database-fundamentals**
  - SQL DDL/DML; joins, subqueries, CTEs, window functions
  - Stored procedures; data cleaning with SQL
  - Python + SQL integration (SQLAlchemy, psycopg2)
  - NoSQL databases: MongoDB, Redis, Cassandra, Neo4j
  - Vector databases for AI applications

- **20-handling-imbalanced-data**
  - Resampling: SMOTE, ADASYN, undersampling, combined strategies
  - Algorithm-level: class weights, threshold tuning, cost-sensitive learning
  - Evaluation: PR-AUC, F1, G-mean; best practices for highly skewed data

- **21-model-explainability**
  - Feature Importance: tree-based, permutation-based
  - SHAP (Tree, Kernel, Deep, Gradient explainers)
  - LIME for local interpretability
  - Partial Dependence Plots (PDP) and ICE plots
  - Regulatory compliance and Responsible AI

## Phase 8: Production & Deployment

**Goal**: Deploy AI models to production with robust infrastructure.

- **13-model-deployment**
  - Model serialization: pickle, joblib, ONNX, TorchScript
  - REST APIs with FastAPI and Flask (type checking, dependency injection, background tasks)
  - Docker for ML; containerization best practices
  - Cloud deployment: AWS SageMaker, GCP Vertex AI, Azure ML, Hugging Face Spaces
  - Production server setup: NGINX, SSL/TLS, rate limiting, authentication
  - A/B Testing: statistical significance, multi-armed bandits, sequential testing
  - Model monitoring: data drift, concept drift, performance degradation

- **14-mlops-basics**
  - Version Control for ML: DVC, MLflow, experiment tracking
  - CI/CD for ML: GitHub Actions, reproducible project templates (Cookiecutter)
  - Model Registry: MLflow, Weights & Biases model artifacts
  - Weights & Biases: hyperparameter sweeps, team collaboration
  - Orchestration: DAG-style pipelines (Airflow, Kubeflow), Kubernetes basics
  - Rollout patterns: canary, blue–green deployments
  - Monitoring & alerting: Prometheus, Grafana
  - Apache Kafka for data streaming; Apache Spark for big data

## Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/road-to-ai-engineer.git
cd road-to-ai-engineer
```

### 2. Set Up Your Environment

#### Option A: Anaconda (Recommended for Beginners)

```bash
# Install Anaconda from https://www.anaconda.com/products/individual

conda create -n ai-engineer-env python=3.10
conda activate ai-engineer-env
pip install -r requirements.txt
```

#### Option B: Python venv

```bash
python -m venv ai-engineer-env

# Activate (Windows)
ai-engineer-env\Scripts\activate

# Activate (Mac/Linux)
source ai-engineer-env/bin/activate

pip install -r requirements.txt
```

### 3. Install Jupyter Notebook

```bash
pip install jupyter notebook
jupyter notebook
```

### 4. Start Learning!

**Quick Start**: Check out [GETTING_STARTED.md](./GETTING_STARTED.md) for a 30-minute first project!

**Full Path**: Follow modules in order:
1. Start with `00-prerequisites`
2. Progress through each phase sequentially
3. Complete exercises and projects
4. Practice, practice, practice!

## Learning Tips

1. **Follow the Order**: Phases are designed to build on each other
2. **Practice Regularly**: Code along with all examples; complete every exercise
3. **Build Projects**: Apply learning through projects — don't just read
4. **Breadth First, Then Depth**: Understand all domains before specializing
5. **Read Papers**: Start reading ML papers from Phase 5 onwards
6. **Experiment**: Break things, ablate, change one variable at a time
7. **Be Patient**: This is the longest path - 25–38 months. Consistency beats intensity
